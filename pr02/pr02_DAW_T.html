Proyecto DAW · 2ª Evaluación

1. Objetivo del proyecto

Diseñar y desplegar en AWS una arquitectura cloud completa que permita:

  • Obtener datos desde una API pública externa.
  • Almacenar los datos en una base de datos en una subred privada.
  • Mostrar los datos mediante una aplicación web accesible desde Internet.

2. Arquitectura mínima obligatoria

2.1 Infraestructura de red

  • VPC propia del proyecto.
  • Subred pública con frontend web e Internet Gateway.
  • Subred privada con microservicio y base de datos.
  • Tablas de rutas configuradas correctamente.
  • Grupos de seguridad diferenciados (frontend, backend, BD).

La base de datos no debe ser accesible desde Internet.

3. Base de datos

  • Ubicada en subred privada.
  • RDS (MySQL/PostgreSQL) o EC2 con motor instalado.
  • Acceso restringido al backend.
  • Al menos un backup o snapshot.

4. Microservicio de ingesta

  • Consumo de API pública externa.
  • Procesado de datos (JSON / CSV).
  • Inserción en base de datos.
  • Ejecución periódica.
  • API key en variables de entorno.
  • Desplegado en subred privada.

5. Aplicación web pública

  • Desplegada en subred pública.
  • Muestra datos reales desde la BD.
  • Acceso vía navegador.
  • Apache o S3.

6. Seguridad y observabilidad

  • Grupos de seguridad mínimos.
  • Gestión correcta de credenciales.
  • Logs básicos.
  • Métrica o alarma en CloudWatch.

7. Entregables

  1. Diagrama de arquitectura.
  2. Código fuente + README.
  3. Capturas comentadas.
  4. Documento técnico (máx. 2 páginas).
  5. URL o IP pública funcional.

8. Normas generales

  • APIs gratuitas.
  • Frecuencia limitada.
  • API distinta por grupo.
  • Copias: calificación 0.

9. APIs autorizadas

Listado idéntico al enunciado oficial.

10. Criterios de evaluación

  • Infraestructura y redes: 25 %
  • Microservicio: 25 %
  • Base de datos: 20 %
  • Seguridad: 15 %
  • Documentación: 15 %
